# 문서 추출 품질 점검 초안

## 1. 목표

- 원문 문서에 존재하는 표, 그림, 도식, 구조 정보가 현재 `processed text` 또는 파싱 결과에서 얼마나 빠졌는지 확인한다.
- 누락 정보가 실제 질의응답 성능에 영향을 주는지 판단한다.
- 우선적으로 개선해야 할 문서를 뽑아 OCR, 구조 추출, 파서 개선 우선순위를 정한다.

## 2. 권장 작업 순서

### Step 1. 판정 기준 먼저 정리

아래 기준을 먼저 고정하고 시작한다.

| 항목 | 판정 기준 |
| --- | --- |
| 표 중요 | 일정, 예산, 평가기준, 제출서류, 자격요건처럼 표에 핵심 정보가 모여 있음 |
| 그림 중요 | 시스템 구성도, 업무 흐름도, 화면 예시, 추진 체계처럼 시각 요소가 핵심 의미를 가짐 |
| 둘 다 중요 | 표와 그림 둘 다 질의응답에 직접 영향을 줌 |
| 낮음 | 본문만으로도 대부분 정보 파악 가능 |
| 누락 | 원문에는 있으나 현재 파싱 결과 또는 추출 텍스트에서 확인되지 않음 |

### Step 2. 후보 문서 10~20개 추리기

다음 특징이 있는 문서를 후보군으로 먼저 모은다.

- 표가 많은 문서
- 일정/예산/평가기준 표가 뚜렷한 문서
- 그림, 도식, 화면 예시가 들어간 문서
- 분량이 길고 구조가 복잡한 문서

### Step 3. 최종 분석 문서 5개 선정

선정 비율 권장안:

- 표 중심 문서 2개
- 그림/도식 중심 문서 2개
- 표와 그림이 모두 중요한 문서 1개

### Step 4. 원문과 처리 결과 비교

문서별로 아래 순서대로 확인한다.

1. 원문에 표/그림/도식이 있는지 확인
2. 파싱 결과나 `processed text`에서 해당 내용이 유지됐는지 확인
3. 빠진 내용이 질의응답에 중요한지 판단
4. 중요도와 개선 필요도를 기록

### Step 5. 문서별 체크리스트 작성

각 문서마다 동일한 형식으로 기록한다.

### Step 6. 대표 예시 4개 정리

- 표 누락 사례 2개
- 그림/도식 누락 사례 2개

### Step 7. 개선 우선순위 확정

- 1순위 문서 3개
- 2순위 문서 5개

## 8. 바로 시작할 때 추천하는 첫 실행안

처음에는 아래처럼 가볍게 시작하는 게 좋다.

1. `files/`에서 표가 많아 보이는 문서 3개, 그림이 많아 보이는 문서 3개를 먼저 고른다.
2. 그중 5개만 최종 선정한다.
3. 각 문서에서 표/그림이 있는 페이지 2~3곳만 먼저 확인한다.
4. `parsed.json` 또는 현재 추출 텍스트와 대조한다.
5. 누락이 명확한 사례를 문서별로 1~2개씩만 적는다.
6. 마지막에 대표 예시 4개와 우선순위를 정리한다.

## 9. 이번 프로젝트에서 특히 볼 포인트

이 프로젝트는 RFP 질의응답이 목적이므로 아래 정보가 빠지면 우선순위를 높게 본다.

- 사업 예산
- 일정과 마감일
- 평가 기준
- 제출 서류와 제출 방식
- 참여 자격
- 시스템 구성도나 추진 체계도
- 화면 예시나 업무 흐름도
